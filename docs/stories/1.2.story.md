# Story 1.2: Color Palette Extraction & Documentation

## Story Statement

**As a** Designer  
**I want to** receive a structured color palette extracted from a website  
**So that** I can document and replicate the brand's color system in a Design System

## Description

Building on the foundation of Story 1.1 (Analysis Engine), this story deepens and professionalizes the color extraction capability. The current `DOMExtractor.extractColorPalette()` returns a flat list of raw CSS color strings — insufficient for a Designer's needs.

Story 1.2 transforms raw color data into a **structured, categorized color palette** with:
- Color normalization (CSS strings → HEX/RGB/HSL)
- Semantic categorization (primary, secondary, accent, neutral, background, text)
- Frequency-based ranking (most-used colors = most important)
- Contrast ratio analysis (accessibility compliance — WCAG 2.1)
- Named color matching (e.g., `rgb(255,0,0)` → `Red`)

### What It Includes

✅ **Color normalization** — CSS color strings → HEX + RGB + HSL  
✅ **Frequency analysis** — rank colors by usage across DOM elements  
✅ **Semantic categorization** — auto-classify into design roles  
✅ **Contrast ratio** — WCAG 2.1 AA/AAA compliance checks  
✅ **Named color matching** — map to nearest named color  
✅ **Palette deduplication** — merge similar colors (tolerance ± 5%)  
✅ **Color documentation output** — structured object ready for report (Story 1.5)  

### What It Does NOT Include

❌ Typography extraction (Story 1.3)  
❌ Spacing/grid analysis (Story 1.4)  
❌ Visual report rendering (Story 1.5)  
❌ UI to display the palette

  - Output per color: `{ hex: "#FF5733", rgb: "rgb(255,87,51)", hsl: "hsl(11,100%,60%)" }`
- [x] Handles invalid/unparseable colors gracefully (skip + log warning)
- [x] Strips alpha channel for solid color comparison (transparent colors noted separately)

### AC 2: Frequency Analysis & Ranking
- [x] Counts how many DOM elements use each color (background + foreground)
- [x] Returns colors sorted by frequency (most used = index 0)
- [x] Minimum frequency threshold: colors used in < 2 elements filtered as noise
- [x] Returns top-N colors (default: 20, configurable)

### AC 3: Semantic Color Categorization
- [x] Automatically assigns one of: `primary`, `secondary`, `accent`, `neutral`, `background`, `text`, `border`, `unknown`
- [x] Categorization rules (heuristic-based):
  - `background`: used as `backgroundColor` on `body`, `html`, `main`, `section` — typically light
  - `text`: used as `color` on `p`, `h1-h6`, `span` — typically dark
  - `primary`: most-used non-neutral non-background non-text color
  - `secondary`: 2nd most-used distinctive color
  - `accent`: high-saturation colors used sparingly (< 10% of elements)
  - `neutral`: near-grayscale colors (saturation < 10%)
  - `border`: colors used exclusively in `border` CSS properties
- [x] Each color tagged with `role: "primary" | "secondary" | ...`

### AC 4: Contrast Ratio Analysis (WCAG 2.1)
- [x] Calculates contrast ratio between:
  - Every `text` color vs every `background` color pair found on same element
  - Primary color vs white and black
- [x] Tags each pair with compliance level:
  - `AAA`: ratio ≥ 7:1
  - `AA`: ratio ≥ 4.5:1 (normal text) / ≥ 3:1 (large text)
  - `FAIL`: ratio < 3:1
- [x] Returns list of failing pairs with recommendation

### AC 5: Color Deduplication (Similarity Merging)
- [x] Merges colors within ± 5% tolerance in HSL space
  - e.g., `#FF5733` and `#FF5A35` → merged as one color (most frequent variant kept)
- [x] Returns both the canonical color and list of merged variants
- [x] Configurable tolerance (default: 5%, range: 1–20%)

### AC 6: Named Color Matching
- [x] Maps each extracted color to the nearest CSS named color
  - e.g., `rgb(255,87,51)` → `{ namedColor: "tomato", distance: 3.2 }`
- [x] Uses Euclidean distance in RGB space for nearest-neighbor match
- [x] Named color list: full CSS4 named colors (147 colors)

### Color Math Reference

**HEX → RGB:**
```javascript
hex.match(/[A-F0-9]{2}/gi).map(v => parseInt(v, 16))
```

**RGB → HSL:**
```javascript
// Standard formula (r,g,b in 0-1 range)
const max = Math.max(r, g, b), min = Math.min(r, g, b);
const l = (max + min) / 2;
const s = max === min ? 0 : l > 0.5
  ? (max - min) / (2 - max - min)
  : (max - min) / (max + min);
```

**WCAG Contrast Ratio:**
```javascript
// Relative luminance L = 0.2126R + 0.7152G + 0.0722B
// Each channel: c <= 0.04045 ? c/12.92 : ((c+0.055)/1.055)^2.4
// Ratio = (L1 + 0.05) / (L2 + 0.05)  where L1 > L2
```

**CSS → parseable:**
- `rgb(r,g,b)` → direct parse
- `rgba(r,g,b,a)` → parse, note alpha
- `#RRGGBB` / `#RGB` → hex parse
- `hsl(h,s%,l%)` → convert to RGB
- Named colors → lookup in CSS4 table

[Source: W3C CSS Color Level 4 / WCAG 2.1 spec]
